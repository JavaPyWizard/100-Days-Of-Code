HTML List Tags

HTML supports three types of lists:

  • Unordered List - <ul>
  • Ordered List - <ol>
  • Description List - <dl>

1. Unordered List <ul>

Used when the order of items doesn’t matter.

Syntax

<ul>
  <li>Item 1</li>
  <li>Item 2</li>
</ul>

Example

  • Frontend
  • Backend
  • Database

2. Ordered List <ol>

Used when the order of items is important.

Syntax

<ol>
  <li>Step 1</li>
  <li>Step 2</li>
</ol>

Example

  1. Install Python
  2. Write Code
  3. Run Script

Ordered List Types

Use type attribute to change marker:

  • type="1" – Numbers (default)
  • type="A" – Uppercase letters
  • type="a" – Lowercase letters
  • type="I" – Upper Roman
  • type="i" – Lower Roman

Example with type="A"

<ol type="A">
  <li>Apple</li>
  <li>Banana</li>
</ol>
  1. Apple
  2. Banana

Example with start and value

<ol start="5">
  <li>Fifth Step</li>
</ol>

<ol>
  <li value="10">Manual Start</li>
</ol>
  1. Fifth Step
  2. Sixth Step
  1. Manual Start
  2. Next Item

3. Description List <dl>

Used to describe terms and definitions.

Syntax

<dl>
  <dt>Term</dt>
  <dd>Description</dd>
</dl>

Example

HTML
HyperText Markup Language
CSS
Cascading Style Sheets

4. Nested List

Syntax

<ul>
  <li>Main
    <ul>
      <li>Sub-item</li>
    </ul>
  </li>
</ul>

Example

  • Frontend
    • HTML
    • CSS
  • Backend
    • Node.js
    • PHP

5. Unordered List Marker Types

● Default (filled circle)

<ul>
  <li>Frontend</li>
</ul>
  • Frontend
  • Backend

○ Hollow Circle — type="circle"

<ul type="circle">
  <li>HTML</li>
</ul>
  • HTML
  • CSS

■ Square — type="square"

<ul type="square">
  <li>Node.js</li>
</ul>
  • Node.js
  • PHP

Common Attributes

Tag Attribute Description
<ol> type Set list style (1, A, a, i, I)
<ol> start Start counting from a specific number
<li> value Set custom number for individual list item
<ul> type Set bullet style (disc, circle, square)

Best Practices

  • ✅ Use unordered lists for unordered data
  • ✅ Use ordered lists for steps or sequences
  • ✅ Use description lists for glossary or FAQ-type data
  • ✅ Avoid over-nesting; max 2–3 levels deep
  • ❌ Don’t use ordered lists when sequence doesn’t matter
